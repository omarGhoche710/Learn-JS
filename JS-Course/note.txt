📝 DOM Attributes vs Properties + data-*

1️⃣ إنشاء element

let div = document.createElement("div");


---

2️⃣ Properties (الأفضل غالبًا ✅)

خصائص جاهزة بالـ DOM

أسرع وأسهل

الأكثر استعمالًا


div.id = "box";
div.className = "card";
div.innerText = "Hello";
div.href = "https://google.com";   // للعناصر المناسبة
div.value = "test";                // input
div.checked = true;                // checkbox

📌 نستعملها لما يكون في property جاهزة


---

3️⃣ Attributes

قيم HTML حرفيًا

للحالات الخاصة


div.setAttribute("id", "box");
div.setAttribute("title", "info");

🚫 createAttribute نادر جدًا استعمالو


---

4️⃣ الفرق المهم 🔥

a.href = "test";              // المتصفح ممكن يعدّل القيمة
a.setAttribute("href", "test"); // بتنحط مثل ما هي بالـ HTML


---

5️⃣ ليش div.data-id ما بيمشي؟ ❌

data-id فيه -

JS ما بتقبل - بأسماء المتغيرات

بتنقري كعملية طرح


div.data - id // ❌


---

6️⃣ الحل الصح: dataset ✅

<div data-id="5" data-user-name="omar"></div>

div.dataset.id = 5;
div.dataset.userName = "omar";

📌 data-user-name ➜ dataset.userName


---

7️⃣ ليش منستعمل data-*؟

✔ نخزّن داتا خاصة بالعنصر
✔ مربوطة بالـ HTML والـ JS
✔ سهلة الوصول من event

مثال:

<button data-product-id="42">Add</button>

btn.onclick = function () {
  console.log(this.dataset.productId);
};


---

8️⃣ ليش ما نستخدم id بدل data-id؟ 🚫

id لازم يكون unique

إلو معنى بالـ DOM

مو لتخزين داتا



---

9️⃣ ليش ما نخزّن الداتا بـ object؟

btn.productId = 42; // ❌ أحيانًا

لأن:

مو ظاهر بالـ HTML

ممكن يضيع مع re-render

frameworks ما تعتمد عليه



---

🔑 الخلاصة النهائية

Property ➜ الخيار الأول

setAttribute ➜ للحالات الخاصة

data- + dataset* ➜ أفضل طريقة لتخزين داتا مرتبطة بالعنصر

createAttribute ➜ نادر جدًا

Do I need to understand only these two things, or is there more detail?

✅ What you need to understand now (the basics)

If you understand these three concepts, you’re already good (about 80%):

1️⃣ Node

👉 The general concept:
Everything in the DOM is a Node.


---

2️⃣ Element

👉 A special type of Node
It represents HTML tags.

<div></div>   <!-- Element Node -->


---

3️⃣ Text

👉 The text inside HTML elements.

<div>Hello</div>

Hello = a Text Node

📌 If these three are clear in your mind, you’re on the right track.


---

⚠️ Are there more details?

Yes, there are other types of Nodes:

Comment Node (<!-- -->)

Document Node

DocumentFragment


❌ But you don’t need them now
❌ You’ll meet them only in more advanced cases


---

💡 How to work in practice (without overthinking)

➤ To create an element:

document.createElement("p"); // Element (Node)

➤ To create text:

document.createTextNode("Hi"); // Text (Node)

➤ To add it to the DOM:

parent.appendChild(node); // node must be a Node


---

🧠 Golden rule

> If a DOM method works with the DOM tree,
it almost always expects a Node.
